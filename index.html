Choose your path

Install a site or build from source.

VonCMS separates runtime users from source users. Site owners start with the Deploy ZIP. Developers fork or clone the repository when they want to study, customize, contribute, or package.

Runtime site owners

Use this path for normal shared-hosting installs.

  • Download the Deploy ZIP from releases.
  • Extract into the hosting web root.
  • Create a MySQL database.
  • Open `/install`, create admin, then publish.
Developers and agencies

Use this path for code, themes, plugins, APIs, and release work.

  • Fork or clone the repository.
  • Install Node dependencies for source work.
  • Run typecheck and build while editing.
  • Use docs for themes, plugins, routing, and security.

FAQ

WordPress alternative questions, answered plainly.

Short answers for publishers comparing VonCMS with traditional CMS and headless stacks.

Is VonCMS a WordPress alternative?

Yes. VonCMS is built for publishers who want normal hosting, a React 19 dashboard, PHP and MySQL runtime, SEO output, ad support, and fewer plugin dependencies.

How much lighter is VonCMS than WordPress?

VonCMS avoids a heavy third-party plugin stack, but exact weight depends on the site, theme, content, and host. Use your own production benchmark before making numeric claims.

Do I need Node.js to run VonCMS?

No. Production hosting does not need Node.js, Vite, npm, or a separate frontend server. Node.js is only needed for source development and custom builds.

Is VonCMS free and open source?

Yes. VonCMS is GPL-3.0-only open-source software with Deploy ZIP releases for runtime installs and source packages for developers.
